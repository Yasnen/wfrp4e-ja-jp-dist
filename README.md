# WFRP4e-ja-JP

Foundry Virtual Tabletop の Warhammer Fantasy Roleplay 4th Edition を日本語化するモジュール（MRyas私家版）

インストール：Foundry Virtual Tabletop（以下FVTT）の設定とセットアップで、アドオンモジュールのモジュールインストールからインストールしてください。

## 注意事項

4.1.0から、「scene.drawings.text」「scene.tokens.name」に対応するため、翻訳データのフォーマットが変更されました。

## 設定方法

1. ローカライズモジュール

    1. モジュールの有効化

        FVTTのゲーム設定「モジュール管理」から「日本語 WFRP4e(MRyas私家版)」と「日本語：FVTT(MRyas私家版)」を有効にする。依存関係のある「Babele」「libWrapper」「fvtt-ja」も合わせて有効化する。

    1. 日本語：FVTT(MRyas私家版)

        FVTTのゲーム設定「設定変更」から「日本語：FVTT(MRyas私家版)」の「言語ファイルフォルダ」を設定する。「デフォルト言語」に「日本語 (MRyas私家版)」を選択すること（未設定の場合ログアウト警告が表示される）。coreモジュール以外でWFRP4e公式モジュールの言語ファイルをローカライズしたファイルを格納した場合やモジュールの言語ファイルを自分で用意した場合など。設定後「変更を保存」して再読込する。

    1. WFRP4e-ja-JP：翻訳データの指定

        「翻訳データ」、「翻訳マージデータ」、「翻訳マージデータフォルダ」を設定する。

    1. Babeleの翻訳ファイル設定

        FVTTのゲーム設定でBabeleの「翻訳ファイルディレクトリ」で指定する。※実翻訳ファイルはこのディレクトリにロケール（日本語の場合「ja」）名のフォルダを作成して格納する必要が有ります。

1. WFRP4eモジュール

    WFRP4e公式モジュールを「モジュール管理」から有効にして読み込む。

## 翻訳データの作成手順（例）

1. モジュールの有効化

    FVTTのゲーム設定「モジュール管理」から対象のWFRP4e公式モジュール、「日本語 WFRP4e(MRyas私家版)」と「日本語：FVTT(MRyas私家版)」を有効にする。依存関係のある「Babele」「libWrapper」「fvtt-ja」も合わせて有効化する。

1. Babeleの翻訳ファイル作成

    翻訳ファイルを作成したい事典(Compendium)を開き、「翻訳」ボタンで翻訳ファイルを出力してください。事典の種類に合わせてファイルを編集する。用意したファイルはBabeleの「翻訳ファイルディレクトリ」で指定するフォルダ配下に日本語を指定する「ja」というフォルダを作成し格納し、再起動してください。

    1. アクター(Actor)

        WFRP4eは「乗り物；Vehicle」が「description」を使うので、以下のマッピングを辞書ファイルに反映した状態で、再度翻訳ファイルを出力することをお勧めします。
```json
"mapping": {
    "biography": "system.details.biography.value",
    "description": "system.details.description.value",
    "gmnotes": "system.details.gmnotes.value",
    "hitloc": "system.details.hitLocationTable.value",
    "species": {
        "path": "system.details.species.value",
        "converter": "actor_species"
    },
    "gender": {
        "path": "system.details.gender.value",
        "converter": "actor_gender"
    },
    "status": {
        "path": "system.details.status.value",
        "converter": "actor_status"
    },
    "haircolour": {
        "path": "system.details.haircolour.value",
        "converter": "actor_color"
    },
    "god": {
        "path": "system.details.god.value",
        "converter": "prayer_god"
    },
    "eyecolour": {
        "path": "system.details.eyecolour.value",
        "converter": "actor_color"
    },
    "items": {
        "path": "items",
        "converter": "items_converter"
    },
    "effects": {
        "path": "effects",
        "converter": "actor_effects_converter"
    },
    "tokenName": {
        "path": "prototypeToken.name",
        "converter": "actor_prototypeToken_name"
    },
    "tokenName": "prototypeToken.name"
}
```
※"tokenName"は「コンバータ」と「パス設定」の２種類あるので、どちらか片方だけ残してください。翻訳ファイルを出力するときに下の「パス設定」のマッピングを使用すると、翻訳ファイルにトークンのプロトタイプのトークン名が出力されます。ローカライズ時に「コンバータ」を使用するとトークンのプロトタイプのトークン名は、翻訳データがある場合は設定された値に、無い場合はアクターの名前のローカライズ結果が反映されます。

    1. アイテム(Item)

    「"entries": {...}」は全て削除する。mappingで追加するコンバータ（翻訳データを参照）で全て動的にローカライズします。

```json
"mapping": {
    "itemName": {
        "path": "name",
        "converter": "itemName"
    },
    "system": {
        "path": "system",
        "converter": "itemSystem"
    },
    "effects": {
        "path": "effects",
        "converter": "itemEffects"
    }
}
```

    1. ジャーナル(Journal)
 ジャーナルのページデータはBabeleの翻訳ファイルには出力されないため、翻訳テンプレートは fvtt-ja のエクスポート機能で取得する。FVTTコンソールで `await FVTTJa_collectJournals()` を実行してジャーナルを収集した後、`FVTTJa_exportJournals("ラベル名")` でJSONをダウンロードできる。詳細は fvtt-ja の README を参照。
```json
"mapping": {
    "pages": {
        "path": "pages",
        "converter": "fvttJaJournalPages"
    }
}
```

    1. シーン(Scene)
```json
"mapping": {
    "drawings": {
        "path": "drawings",
        "converter": "sceneDrawingsJP"
    },
    "notes": {
        "path": "notes",
        "converter": "sceneNotesJP"
    },
    "tokens": {
        "path": "tokens",
        "converter": "sceneTokensJP"
    }
}
```

    1. ロール表(Table)
```json
"mapping": {
    "results": {
        "path": "results",
        "converter": "tableResultsJP"
    }
}
```

1. 翻訳データ作成

    翻訳ファイルで呼び出されるBabeleのコンバータで、翻訳データが生成されます。漏れなく生成するため、ローカライズする事典すべてをワールドにインポートしてください。WFRP4eシステムのインストールでは一部インポートされません（例：ほぼすべてのモジュールのアイテム、コアモジュールの怪物寓話集など）。インポートが完了後「game.wfrp4ejajp.exportDict()」でエントリが追加された翻訳データをダウンロードできます。ダウンロードしたjsonファイルの値を適切に日本語へ翻訳してください。

1. 翻訳データの設定

    前項で作成した翻訳データは、「日本語 WFRP4e(MRyas私家版)」の設定「翻訳データ」で指定すると、標準で用意されている翻訳データの代わりに使用することが出来ます。公式モジュール毎に「翻訳データ」を分割する場合、「翻訳マージデータ」や「翻訳マージフォルダ」で追加読込することも可能です。

## その他

### 注意点
* 本モジュールはFVTTで動作するWFRP4eシステムを日本語化するモジュールですが、ホビージャパン社（以下HJ社）とは全く関係ありません。本モジュールについて、HJ社にお問い合わせするなどの行為は、**絶対にお止め**ください。
* オンセ工房では動作しません。FVTTコアのデフォルト言語設定には、拙作モジュールの「fvtt-ja」をお使いください。
* 既存の wfrp4e-jp とは別のモジュールです。

### WFRP4eシステム起因の課題
* Babeleで書き換えて動作するようにに設定していますが、不具合有ればご連絡ください。
* Compendiumに異なるTypeで同名のItemが存在したり、Actorに組み込まれたItemや追加されたモジュールで同名のItemが異なるプロパティを持つため、ローカライズはItem名だけではなく、プロパティの値に応じてローカライズ結果を指定できるようにしました。これはBabeleの標準的な使い方とは思えないため、不具合が出る可能性が有ることをご了承ください。
* 上述の仕組みのため、Babeleのバージョンアップで動作しなくなる可能性があるのでご注意ください。
* WFRP4eシステムでローカライズの仕組みが用意されていない情報は、毎回読み込まれたデータを上書きする形で対応しています（正しい方法が解らないので暫定対応）。FVTT既定の設定では「プレイヤー」は権限が足りないため動作しません。「信頼済みプレイヤー」にする必要が有ります。

### 未翻訳部分について
#### 日本語翻訳権への対応
事典の説明などのゲームの根幹、ゲームシステムなどに深くかかわる部分は意図的に訳しておりません。日本国内ではHJ社が発行しているウォーハンマーRPG ルールブック（以下HJルルブ）があり、日本国内での翻訳権などはHJ社が所有していると思われ、その権利を侵害する懸念があるためです。未訳部分はHJルルブを参照しながらお使いいただくか、ご自身でBabeleの翻訳ファイルや辞書データを編集することで日本語化してください。
#### 対応漏れや対応が困難なもの
FVTTやWFRP4eシステムの制限や未対応という理由でローカライズされていない項目があります。後者は修正依頼もできますが公開されたリポジトリを元に修正することも可能です。一部を以下に示します。
- アイテム（武器）の一部プロパティ（ダメージ）で参照する能力値。
- テストの内訳を示すツールチップでユーザの手入力を示すコメントが「User Entry」
- 堕落への露出処理、クリーチャー特徴で「堕落」を保有する相手と戦闘で遭遇した後、堕落への露出が処理出来ますが、堕落の規模が日本語化されていると、現状では動作しません
- キャラジェネの「詳細の決定」で「名前」生成は対応していません。英語名を生成する場合、「名前」の生成ボタンを押下するときは性別に「Male/Female」を入力しておいてください

### ご自分でさらに日本語化を実施したい場合。
本モジュールに含まれているjsonファイルを編集することで、ほぼ100%ローカライズすることが可能です。本モジュールを更新すると上書きされてしまうため、別の場所に保管して編集することをお勧めします。
- compendium/*.json
Babeleの翻訳ファイル。別の場所に格納する場合、その場所をBabeleの設定で「翻訳ファイルディレクトリ」に設定してください。「Actor」「Item」「Scene」「Table」のCompendiumについては、コンバータが用意されています、公式モジュールを追加でローカライズしたい場合種類に応じた「mapping」をコピー＆ペーストしてください。「Item」については名前も動的にコンバータで処理するため「entries」が不用です。
- json/wfrp4e-ja-jp.json
上述したBabele用のコンバータが参照するデータが格納されていますが、全てのデータが登録されていません。ゲーム中にマクロで「game.wfrp4ejajp.exportDict("wfrp4e-ja-jp.json");」（ファイル名は省略可）を実行すると、ローカライズ可能な項目が追加されたzip圧縮されたjsonをダウンロードすることが可能です。このjsonファイルを編集し、「日本語WFRP4e（MRyas私家版）」の設定「翻訳データ」で指定すると読み込むことが可能です。
- 翻訳マージデータ、前項の翻訳データは分割して追加読込が出来ます。「翻訳マージデータ」で１ファイル、「翻訳マージデータフォルダ」でフォルダを指定すると、読み込み追加・上書きされます。
- lang/wfrp4e-core-ja.json
WFRP4e - Core Rulebook Moduleの日本語言語ファイルですが、キャラクタ作成が正常に動作するよう、所持品の項目だけをローカライズしてありますが、fvtt-jaの「独自言語ファイル」機能を使うことで、ご自身でローカライズすることも可能です。「modules/wfrp4e-core/lang/en.json」を編集、「独自言語ファイル」で設定したフォルダに「wfrp4e-core-ja.json」というファイル名で格納すると日本語化されます。
